# 数据库迁移助手

此技能帮助管理学校管理系统的数据库迁移和模式变更。它为创建、应用和回滚数据库迁移提供指导和自动化，确保不同环境之间的数据库模式一致性。

## 功能特性

- 为模式变更生成数据库迁移脚本
- 将迁移应用到不同环境
- 在需要时回滚迁移
- 跟踪迁移历史和状态
- 处理数据库模式冲突
- 支持常见的数据库操作（添加列、重命名表等）
- 与现有的SQLAlchemy模型集成
- 提供迁移回滚能力

## 使用方法

### 基本使用

要管理数据库迁移：

1. 指定迁移操作（创建、应用、回滚、状态）
2. 定义模式变更或选择现有迁移
3. 选择目标环境（本地、staging、生产）
4. 执行迁移操作
5. 验证迁移结果

### 迁移操作

#### 创建迁移

基于模式变更创建新的迁移脚本：

1. 检测SQLAlchemy模型中的变更
2. 生成带有升级和降级函数的迁移脚本
3. 如有需要，审查并编辑迁移脚本
4. 将迁移脚本保存到迁移目录

#### 应用迁移

将待处理的迁移应用到数据库：

1. 识别待处理的迁移
2. 按时间顺序应用迁移
3. 更新迁移历史
4. 验证迁移后的数据库模式

#### 回滚迁移

回滚最后应用的迁移：

1. 识别最后应用的迁移
2. 运行迁移的降级函数
3. 更新迁移历史
4. 验证回滚后的数据库模式

#### 迁移状态

检查迁移状态：

1. 比较已应用的迁移与可用的迁移
2. 识别待处理和已应用的迁移
3. 显示迁移历史和时间戳
4. 验证数据库模式与迁移状态匹配

### 生成的文件

该技能将生成或更新以下文件：

- **迁移脚本**：`migrations/versions/{timestamp}_{description}.py`
- **迁移配置**：`migrations/alembic.ini`
- **迁移环境**：`migrations/env.py`
- **迁移历史**：在数据库中跟踪迁移

## 配置选项

### 迁移设置

- **迁移目录**：指定存储迁移脚本的位置
- **数据库URL**：配置数据库连接字符串
- **迁移格式**：选择迁移脚本格式（alembic、原始SQL）
- **版本方案**：配置如何跟踪迁移版本

### 环境配置

- **本地**：为本地开发数据库配置
- **Staging**：为staging环境数据库配置
- **生产**：为生产环境数据库配置

### 迁移选项

- **自动生成**：从模型变更自动生成迁移
- **手动编辑**：允许手动编辑迁移脚本
- **批处理模式**：批量应用多个迁移
- **试运行**：预览迁移而不应用它们
- **详细输出**：显示详细的迁移执行信息

## 最佳实践

1. **版本控制**：在版本控制中存储迁移脚本
2. **测试**：在生产前在staging环境测试迁移
3. **备份**：在应用迁移前创建数据库备份
4. **文档**：记录重大模式变更
5. **回滚计划**：始终为迁移准备回滚计划
6. **增量变更**：进行小的、增量的模式变更
7. **数据迁移**：与模式变更一起处理数据迁移
8. **性能考虑**：为大型数据库优化迁移

## 故障排除

### 常见问题

- **迁移冲突**：解决多个迁移脚本之间的冲突
- **模式不匹配**：修复模型与数据库模式之间的不一致
- **数据丢失**：从迁移过程中的意外数据丢失中恢复
- **连接问题**：排查数据库连接问题
- **权限错误**：解决迁移过程中的数据库权限问题

### 解决步骤

1. **迁移冲突**：合并冲突的迁移或重新创建它们
2. **模式不匹配**：使用`stamp`命令设置迁移状态
3. **数据丢失**：从备份恢复并重新应用迁移
4. **连接问题**：验证数据库连接字符串和网络访问
5. **权限错误**：确保数据库用户具有所需的权限

## 与开发工作流集成

### 本地开发

1. 对SQLAlchemy模型进行更改
2. 生成并审查迁移脚本
3. 将迁移应用到本地数据库
4. 使用新模式测试应用功能

### Staging环境

1. 将代码更改合并到staging分支
2. 将迁移应用到staging数据库
3. 运行集成测试
4. 验证应用功能

### 生产部署

1. 创建数据库备份
2. 将迁移应用到生产数据库
3. 监控应用性能
4. 准备好回滚计划，以防出现问题

数据库迁移助手为学校管理系统中的数据库模式变更管理提供了全面的解决方案。通过自动化迁移创建和执行，它确保了跨环境的一致数据库模式，并降低了模式相关问题的风险。此技能与开发工作流无缝集成，为应用程序增长时的数据库模式演变提供了可靠的方法。
