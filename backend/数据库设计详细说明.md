# 数据库设计详细说明

## 1. 数据库表结构

### 1.1 students 表

| 字段名         | 数据类型        | 约束                         | 描述                                                        |
| :---------- | :---------- | :------------------------- | :-------------------------------------------------------- |
| student\_id | VARCHAR(20)  | PRIMARY KEY                | 学号，格式为 "STU2024xxxx"                                      |
| name        | VARCHAR(50)  | NOT NULL                   | 学生姓名                                                      |
| gender      | VARCHAR(10)  | NOT NULL                   | 性别                                                        |
| class       | VARCHAR(20)  | NOT NULL                   | 班级，格式为 "1班"、"2班" 等                                        |
| grade       | VARCHAR(20)  | NOT NULL                   | 年级，只能是 "高一"、"高二"、"高三"                                     |
| contact     | VARCHAR(100) | <br />                     | 联系方式                                                      |
| status      | VARCHAR(20)  | NOT NULL                   | 状态，可选值：active(在校)、suspended(休学)、graduated(毕业)、dropped(退学) |

### 1.2 teachers 表

## 3. 表关系

### 3.1 教师与班级关系

- `teachers` 表与 `teacher_classes` 表：一对多关系，一个教师可以任教多个班级
- `teachers` 表与 `homeroom_teachers` 表：一对多关系，一个教师可以担任多个班级的班主任

### 3.2 课程相关关系

- `courses` 表与 `student_courses` 表：一对多关系，一个课程可以有多个学生课程安排
- `courses` 表与 `teacher_courses` 表：一对多关系，一个课程可以有多个教师课程安排

### 3.3 学生相关关系

- `students` 表与 `student_grades` 表：一对多关系，一个学生可以有多个成绩记录

### 3.4 教师与课程关系

- `teachers` 表与 `student_courses` 表：一对多关系，一个教师可以有多个学生课程安排
- `teachers` 表与 `teacher_courses` 表：一对多关系，一个教师可以有多个教师课程安排
- `teachers` 表与 `teaching_progress` 表：一对多关系，一个教师可以有多个教学进度记录

### 3.5 考试相关关系

- `exams` 表与 `student_grades` 表：一对多关系，一个考试可以有多个成绩记录

## 4. 特殊字段说明

### 4.1 学号格式

- 格式："STU2024xxxx"
- 其中 "STU" 为前缀，"2024" 为年份，"xxxx" 为四位数字

### 4.2 工号格式

- 格式："TEA2024xxxx"
- 其中 "2024" 为年份，前两位 "xx" 为学科代码，后两位 "xx" 为编号
- 学科代码：01-语文，02-数学，03-英语，14-物理，15-化学，16-生物，24-历史，25-政治，26-地理，21体育，22音乐

### 4.3 班级格式

- 格式："1班"、"2班" 等
- 与年级组合使用，如 "高一1班"

### 4.4 年级选项

- 只能是 "高一"、"高二"、"高三"

### 4.5 职称选项

- 只能是 "高级教师"、"一级教师"、"二级教师"

## 5. 数据状态说明

### 5.1 学生状态

- active：在校
- suspended：休学
- graduated：毕业
- dropped：退学

### 5.2 考试状态

- 准备中

### 5.3 教学进度状态

- not-started：未开始

### 5.4 教室状态

- available：可用

### 5.5 课程状态

- active：活跃

## 6. 数据库设计特点

1. **完整性约束**：使用外键约束确保数据一致性
2. **时间戳**：关键表使用时间戳记录创建和更新时间
3. **默认值**：为常用字段设置合理的默认值
4. **枚举值**：使用常量定义有效选项，确保数据有效性
5. **关系清晰**：通过外键和关系定义，确保表之间的关联清晰
